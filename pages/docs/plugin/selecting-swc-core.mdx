# Selecting the version

Currently, the Wasm plugins are not backwards compatible.
So you need to select an appropriate version of `swc_core` for your plugin.

Thie page documents the version of `swc_core` you can use for each runtime.

## swc_core

If you use a version in a range of the title, the Wasm plugin will work for the runtimes written in the body.

### `swc_core@0.69.x` ~

- [`@swc/core@1.3.40`](https://github.com/swc-project/swc/releases/tag/v1.3.40) ~
- [`next@13.2.5-canary.5`](https://github.com/vercel/next.js/releases/tag/v13.2.5-canary.5) ~

### `swc_core@0.66.x` ~ `swc_core@0.68.x`

- [`@swc/core@1.3.39`](https://github.com/swc-project/swc/releases/tag/v1.3.39)

### `swc_core@0.61.x` ~ `swc_core@0.64.x`

- [`@swc/core@1.3.38`](https://github.com/swc-project/swc/releases/tag/v1.3.38)

### `swc_core@0.54.x` ~ `swc_core@0.59.x`

- [`@swc/core@1.3.29`](https://github.com/swc-project/swc/releases/tag/v1.3.29) ~ [`@swc/core@1.3.37`](https://github.com/swc-project/swc/releases/tag/v1.3.37)

- [`next@13.2.4-canary.0`](https://github.com/vercel/next.js/releases/tag/v13.2.4-canary.0) ~
